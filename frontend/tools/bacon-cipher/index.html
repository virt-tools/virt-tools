Bacon's Cipher

Encode and decode with Bacon's cipher, the steganographic cipher devised by Francis Bacon around 1605. Each letter of the alphabet is represented by a group of five symbols drawn from a two-letter alphabet (historically A/B, equivalently 0/1) — so a message becomes a binary stream that can be hidden in anything with two states (two typefaces, bold/regular, left/right, you name it). Choose the modern 26-letter alphabet (I, J, U, V all distinct) or Bacon's original 24-letter alphabet (where I/J and U/V share a code). Encode text to symbol groups and decode a symbol stream back to text, with the full code table shown for reference. Everything runs locally in your browser.

The math. Bacon's cipher is a 5-bit binary encoding of the alphabet. In the 26-letter variant, each letter's position 0–25 (A=0 … Z=25) is written as a 5-bit binary number, most-significant bit first, with each bit rendered as A (0) or B (1) — so A=AAAAA, B=AAAAB, Z=BBAAB. Bacon's original used a 24-letter alphabet (I/J shared a code and U/V shared a code), so positions run only 0–23 and Z=BABBB. Encoding maps each plaintext letter (non-letters ignored) to its 5-symbol group; decoding collects the symbol stream (any characters other than A/B/0/1 are ignored, and 0≡A, 1≡B) into 5-bit groups and maps each value back to a letter, marking values outside the alphabet range (24–31 for the 24-letter variant, 26–31 for the 26-letter) as ?. Historically the real trick wasn't the encoding but the hiding: Bacon concealed the A/B stream as two subtly different typefaces in an innocent-looking cover text.
