Columnar Transposition Cipher

Encrypt and decrypt with the columnar transposition cipher, a classical cipher that rearranges (rather than substitutes) letters. Pick a keyword whose length sets the number of columns; the keyword's alphabetical order sets the order in which columns are read. To encrypt, the plaintext (letters only, upper-cased) is written row by row into a grid of that width and padded to fill the last row; the ciphertext is then read column by column, in the keyword's rank order. To decrypt, the process is inverted: the ciphertext is split back into columns of the right lengths (the first n mod width columns are one longer) and read off row by row. Enter a keyword and your text to encipher or decipher, with the column order shown for reference. Pairs with the Rail Fence cipher in the Classical Ciphers tool, and with the Caesar and Vigenère ciphers. Everything runs locally in your browser.

Column order for keyword

The math. For a keyword of length w, build a rank for each column by sorting the keyword's letters alphabetically, breaking ties by original position (so duplicate letters keep their left-to-right order). Column i gets the rank rᵢ = its index in that sorted order (0-based). To encrypt, write the upper-cased, letters-only plaintext into a w-column grid row by row, padding the final row with X, then emit the columns sorted by rank ascending — each read top to bottom. To decrypt, note that if the ciphertext length is n, the grid has ceil(n/w) rows and the first n mod w columns (in original order) are one row taller; slice the ciphertext into those column lengths in rank order, refill the grid by column, and read it off row by row, stripping the trailing X padding. Transposition ciphers keep each letter's identity but hide its position, so they're weak alone but historically strong when combined with a substitution cipher (e.g. ADFGVX). Pairs with the Classical Ciphers, Caesar, and Vigenère tools. Everything runs locally — nothing leaves your browser.
